Claude Code 修复后台代理、MCP 与权限交互。

Claude Code fixes background agents, MCP, and permission flows.

Claude Code v2.1.141 是一次偏工作流稳定性的版本：hooks、plugins、background agents、MCP、Windows/WSL、Bedrock/Vertex 等路径都有修复，适合重度用户尽快升级。

  • hooks 新增 terminalSequence，可以在没有 controlling terminal 的情况下发桌面通知、窗口标题或 bell，适合把长任务完成、测试失败、需要人工确认做成可见信号。
  • CLAUDE_CODE_PLUGIN_PREFER_HTTPS 让 GitHub plugin 源走 HTTPS clone，企业机器或 CI 没配 GitHub SSH key 时更省事。
  • ANTHROPIC_WORKSPACE_ID 支持 workload identity federation 场景下把 minted token 限定到具体 workspace，适合多 workspace 的企业身份规则。
  • claude agents --cwd <path> 可以按目录过滤 session 列表，适合 monorepo 或多项目并行开发。

  • /feedback 可附带最近 24 小时或 7 天 session，跨 session 的 bug 更容易报清楚。
  • /bg 和 ←← 启动的 background agents 会保留当前 permission mode，不会回到默认权限，减少后台任务权限漂移。
  • 多个烦人的交互问题被修复：/model 不再意外改变其他并发 session 的 autocompact threshold；权限弹窗打开时按 Enter 不再同时提交输入框；vim INSERT/VISUAL 下 Ctrl+C 可以中断运行 turn。

Claude Code v2.1.141 is mostly a workflow-stability release: hooks, plugins, background agents, MCP, Windows/WSL, and Bedrock/Vertex paths all received fixes, so heavy users should prioritize the upgrade.

  • Hooks now support terminalSequence, allowing desktop notifications, window titles, or bells even without a controlling terminal. This is useful for surfacing long-task completion, test failures, or human-approval waits.
  • CLAUDE_CODE_PLUGIN_PREFER_HTTPS makes GitHub plugin sources clone over HTTPS, which helps enterprise machines or CI environments without a GitHub SSH key.

mi v1.7.5：给极简本地 agentic loop 加上 Docker 沙箱mi v1.7.5 Adds Docker Sandboxing to a Tiny Local Agent Loop

本地 agent 跑命令时应默认进 Docker 沙箱。

Local agents that run commands should default to Docker isolation.

mi v1.7.5 给一个极小 agentic loop 加了 --sandbox，把 agent 放进 Docker 并挂载当前目录；它的价值不在功能复杂，而在提醒我们：本地 agent 默认就应该隔离运行。

  • mi 适配 OpenAI-compatible model，包括本地模型；这类轻量 agent 很适合做“修测试直到通过”“批量小改”“探索代码并给 patch”这种窄任务。
  • --sandbox 用 Docker 限制意外命令的影响范围，尤其适合让 agent 跑未知测试脚本、安装依赖、执行项目自带工具链。
  • 当前目录会被挂载，意味着代码仍可能被修改；隔离不是备份。执行前先建分支或 worktree，执行后用 git diff 审查。
  • 对 Claude Code / Codex 用户的迁移价值：即使不用 mi，也应该把“高权限 agent 任务”改成在容器、devcontainer、Docker sandbox 或临时 worktree 里运行。

最小用法：

mi --sandbox -p 'fix tests until all are passing'

更稳的本地工作流：

git switch -c ai/fix-tests
mi --sandbox -p 'Run the test suite, fix the smallest necessary code changes, and stop when all tests pass. Do not change public APIs unless tests prove it is required.'

- 不要重构无关代码
- 测试全部通过后停止
- 最后输出修改文件、失败根因、验证命令
  • 现在能做的一步：给所有“让 agent 自己跑命令”的任务默认加 git switch -c ai/<task> 或 git worktree add，并优先放进 Docker/devcontainer 执行。

mi v1.7.5 adds --sandbox to a tiny agentic loop, running the agent inside Docker with the current directory mounted. Its value is less about feature depth and more about the default: local agents should run isolated.

  • mi works with OpenAI-compatible models, including local models. Lightweight agents like this are a good fit for narrow tasks such as “fix tests until green,” small batch edits, or code exploration that produces a patch.
  • --sandbox uses Docker to reduce the blast radius of unexpected commands, especially when the agent runs unknown test scripts, installs dependencies, or invokes project tooling.
  • The current directory is mounted, so the code can still be modified. Isolation is not backup. Create a branch or worktree first, then inspect with git diff afterward.
  • The lesson for Claude Code / Codex users: even if you do not use mi, high-permission agent tasks should run inside a container, devcontainer, Docker sandbox, or temporary worktree.
