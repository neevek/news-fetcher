Sieve 提醒的新基线：把 AI 聊天存储纳入密钥泄露扫描Sieve Highlights a New Baseline: Scan AI Transcript Storage for Secret Leaks

把 AI 聊天记录纳入密钥扫描与轮换流程。

Add AI transcript files to your secret-scanning and key-rotation workflow.

这是一个面向 macOS 的本地安全工具，专门扫描 Cursor/Claude 等 AI 编程工具的聊天与状态存储，帮助你找出被意外落盘的密钥；对 Claude Code/Codex 用户来说，核心价值是把“提示词泄露秘密”从抽象风险变成可检查、可修复的流程。

  • 先建立一个现实假设：AI 代理读取过的 .env、config、命令输出，都可能进入本地 transcript/SQLite/state 文件，且常常不在 git 追踪范围内但长期存在。
  • 把“代码仓库 secret scan”升级为“开发环境 secret scan”：不仅扫项目目录，也扫 AI 工具本地数据目录（如 ~/Library/Application Support/...）。
  • 在日常流程中增加一个“会话后清理”步骤：
    • 轮换（rotate）本次会话接触过的高权限 key。
    • 删除或归档本地会话数据库/日志中包含敏感信息的片段。
    • 重新生成最小权限 token（只开必要 scope）。
  • 给 AI 编程会话加最小暴露策略：
    • 把真实密钥放在独立进程注入，不直接粘贴到对话。
    • 用假值/占位符让模型先完成代码骨架，再在本地替换。
  • 可直接落地的扫描命令（先手工审计，再接入脚本）：
# 1) 扫当前仓库
rg -n --hidden --no-ignore -S "(sk-|api[_-]?key|access[_-]?token|-----BEGIN)" .

# 2) 扫常见 macOS AI 工具数据目录（按你实际安装调整）
rg -n -S "(sk-|api[_-]?key|access[_-]?token|-----BEGIN)" \
  "$HOME/Library/Application Support/Cursor" \
  "$HOME/Library/Application Support/Claude" 2>/dev/null
  • 可直接套用的 .gitignore 补充（减少二次扩散）：
.env
.env.*
*.pem
*.key
*.p12
secrets.*
  • 团队实践建议：把“本地 transcript secret 扫描”加入每周安全例行检查，和依赖漏洞扫描同级对待。

This is a local macOS security utility that scans Cursor/Claude-style chat and state storage for leaked secrets; for Claude Code/Codex users, the key value is turning “prompt-time secret leakage” into a concrete, fixable workflow.

  • Start with a realistic assumption: anything an AI agent reads from .env, config, or command output can end up in local transcripts/SQLite/state files and persist for a long time.
  • Upgrade from “repo secret scanning” to “workstation secret scanning”: scan both project files and local AI app data directories (for example ~/Library/Application Support/...).

    • 在防火墙/VPN/公司网络不稳定场景下，CLI 可用性明显提升。
  • 终端显示稳定性修复（窗口尺寸事件丢失、超长会话字形污染、VS Code 渲染问题）：
    • 长时间 Pairing 或代理长跑任务时，不再频繁依赖 Ctrl+L 或重启。
