可以马上把 live agents 接到状态栏或脚本里：

# 查看当前所有 Claude session 的机器可读状态
claude agents --json | jq .

# 统计等待输入的 agent 数量
claude agents --json \
  | jq '[.[] | select(.awaiting_input == true)] | length'

# 列出需要你处理的 session
claude agents --json \

#!/usr/bin/env bash

fi

AI 聊天记录也要纳入 secret scanning。

AI chat history should be part of secret scanning.

Sieve关注的是一个经常被忽略的 AI 编程安全面：Cursor、Claude Code、Copilot、Cline 等工具读过 .env 后，密钥可能进入本地聊天记录、SQLite 状态库或工作区缓存。对 Claude Code / Codex 用户来说，这提醒你要把“AI transcript secret scanning”加入本地安全习惯，而不只是扫 Git 仓库。

  • .gitignore 只能防止文件进 Git，不能防止 agent 把密钥读进 transcript、tool log、workspace storage 或 crash/debug 文件。
  • 当你让 AI 帮忙接 OpenAI、Stripe、AWS、GitHub App、数据库连接时，优先提供变量名和配置结构，不要直接粘贴真实 secret。
  • 把真实密钥放进系统 secret manager、CI secret、direnv、1Password CLI、AWS Secrets Manager 等工具，让 agent 只看到占位符。
  • 如果 agent 已经读过 .env，要把对应 key 当作已暴露处理：先 rotate，再清理本地缓存，再补上规则防止再次读取。
  • 扫描范围应该包括 IDE 和 agent 的本地目录，而不只是项目目录。

可以先用开源工具做一次本地粗扫。macOS 上常见目录示例：

# 安装扫描工具（二选一或都装）
brew install gitleaks trufflehog

# 扫当前仓库
gitleaks detect --source . --no-git --redact
trufflehog filesystem . --only-verified

# 扫常见 AI 编程工具的本地状态目录
for dir in \
  "$HOME/Library/Application Support/Cursor" \
  "$HOME/Library/Application Support/Claude" \
  "$HOME/.claude" \
  "$HOME/.codex" \
  "$HOME/Library/Application Support/Code"; do
  [ -d "$dir" ] && gitleaks detect --source "$dir" --no-git --redact || true
done

更关键的是从工作流上减少 secret 暴露。可以给项目加一条 agent 规则：

# AI secret handling rule
- Do not read `.env`, `.env.*`, private key files, or credential exports unless explicitly asked.
- When code needs a secret, use environment variable names such as `OPENAI_API_KEY` and placeholders like `<redacted>`.
- Never paste, print, summarize, or store real secret values in chat, logs, generated docs, tests, or examples.

让 AI 改配置时，可以这样提示：

请实现 OpenAI API 配置读取，但不要读取或打印 `.env`。
只使用环境变量名 `OPENAI_API_KEY`，示例值必须写成 `<redacted>`。
如果需要说明本地配置步骤，只给出 shell export 模板，不包含真实 key。

如果扫描命中真实 key，处理顺序要快：revoke/rotate key -> invalidate sessions/tokens -> 清理本地 transcript/cache -> 补 secret handling rule -> 重新扫描确认。

Sieve focuses on an AI coding security issue that is easy to miss: after tools like Cursor, Claude Code, Copilot, or Cline read .env, secrets can end up in local chat logs, SQLite state databases, or workspace caches. For Claude Code and Codex users, the lesson is to add “AI transcript secret scanning” to local security hygiene, not just scan Git repositories.

  • .gitignore only keeps files out of Git; it does not stop an agent from copying secrets into transcripts, tool logs, workspace storage, or crash/debug files.
  • When asking AI to wire up OpenAI, Stripe, AWS, GitHub App, or database credentials, provide variable names and config structure instead of real secrets.
  • Put real keys in a system secret manager, CI secrets, direnv, 1Password CLI, AWS Secrets Manager, or similar tooling, so the agent sees only placeholders.
  • If an agent has already read .env, treat the corresponding key as exposed: rotate it, clean local caches, then add rules to prevent repeat exposure.
  • Scan IDE and agent local directories, not just the project directory.
