Claude Code RCE 复现：常见 agent 工具执行模式存在安全风险I reproduced a Claude Code RCE. The bug pattern is everywhere

RCE 复现提醒要收紧 agent 工具执行权限。

这篇文章复现 Claude Code RCE，并指出相关 bug pattern 很常见；对所有 agentic coding 用户，重点是重新审视“模型输出到工具执行”的安全边界。

  • 高风险模式通常出现在：未审查的命令执行、自动读取不可信文件、hook/MCP/tool 把外部输入直接传给 shell。
  • 不要让 agent 在默认状态下拥有无限制 shell、网络、凭据和写权限；优先使用 allowlist、确认提示和最小权限目录。
  • 对开源 repo、issue、PR、日志、网页内容要按不可信输入处理，因为它们可能包含 prompt injection 或命令诱导。
  • 团队应审计 .claude、MCP server、hooks、CI agent 配置，确认没有把用户输入拼接进危险命令。
  • 现在可做的一步：把 Claude Code 的高风险操作切到手动确认，并在容器或临时 workspace 中处理陌生代码。
