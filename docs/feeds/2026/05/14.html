OpenAI 内部 Codex 安全实践：可复用的企业治理模板OpenAI’s Codex Safety Setup Is a Practical Enterprise Governance Template

Codex 企业落地的安全配置可以直接照抄。

Codex’s enterprise safety setup is directly reusable.

OpenAI 公开了内部如何用沙箱、审批、网络策略、身份绑定和 OpenTelemetry 运行 Codex；对工程团队来说，这是一份可以直接改造成企业 AI 编程治理基线的配置清单。

  • 把安全策略拆成两层：sandbox 负责技术边界，approval policy 负责哪些动作必须停下来让人确认。
  • 给日常开发目录写入权限，但限制 Codex 的运行模式，避免 agent 随意写到工作区外。
  • 网络访问不要默认全开：允许常用域名，阻断高风险域名，对陌生域名要求审批。
  • 凭据放 OS keyring，并把登录方式和 workspace 绑定到企业账号，减少个人 token、错误 workspace、MCP 凭据泄露带来的风险。
  • 用 rules 把命令分级：git status、gh pr view、kubectl logs 这类只读调试命令可以放行；部署、删除、写生产资源的命令要求审批或禁止。
  • 开启 OTel 日志，把 prompt、工具调用、审批决策、MCP 使用、网络拦截事件送进 SIEM，出问题时能追溯“用户想做什么”和“agent 实际做了什么”。

OpenAI described how it runs Codex internally with sandboxing, approvals, network policy, identity binding, and OpenTelemetry. For engineering teams, it is a practical baseline for governing AI coding agents without stopping everyday development.

  • Split safety into two layers: sandbox defines the technical boundary, while the approval policy decides which actions must pause for human confirmation.
  • Grant write access to normal development directories, but restrict the allowed Codex sandbox modes so the agent cannot casually write outside the workspace.
  • Do not leave network access wide open: allow common domains, block risky domains, and require approval for unfamiliar destinations.
  • Store credentials in the OS keyring, and bind login plus workspace to the enterprise account to reduce personal-token, wrong-workspace, and MCP credential risks.
  • Classify shell commands with rules: allow read-only commands such as git status, gh pr view, and kubectl logs; require approval or denial for deployment, deletion, and production-write commands.
  • Enable OTel logs so prompts, tool calls, approval decisions, MCP usage, and network allow/deny events flow into your SIEM. When something goes wrong, you can reconstruct both user intent and agent behavior.

Codex 进入 ChatGPT 手机端：把手机变成编程代理的远程控制台Codex in ChatGPT Mobile Turns the Phone into a Remote Control for Coding Agents

手机可远程监督 Codex 长任务。

Use your phone to supervise long-running Codex work.

Codex 进入 ChatGPT 手机端后，AI 编程代理不再只适合坐在电脑前使用，而是可以把长任务交给桌面端执行、用手机处理关键决策。对 Claude Code / Codex 用户来说，重点不是“在手机上写代码”，而是把手机变成远程审阅、授权和纠偏入口。

  • 适合处理长时间任务：让 Codex 在 Mac、devbox 或远程环境里跑测试、改代码、生成 diff；你在路上只处理 approve、补充上下文、选择方案。
  • 本地开发环境仍留在电脑上：代码、credentials、permissions、依赖和本地配置不需要搬到手机端，手机只接收实时状态、terminal output、screenshots、diffs、test results 和 approvals。
  • 可以把“等待 AI 完成”改成异步工作流：早上发起 bug 排查，中途手机确认方向，回到电脑时直接 review patch。
  • 对 Claude Code 用户也有借鉴价值：把任务拆成需要低频人工判断的步骤，并在提示词里明确“遇到不确定分支时暂停并给出选项”，更适合远程跟进。
  • 团队要提前收紧权限边界：既然手机可以批准命令，建议在 Codex / Claude Code 项目里把危险命令、secret 读取、部署动作做成显式审批点。

可以直接照做的移动端任务模板：

你在我的本地开发环境中工作。请排查 issue #123，但先不要提交或部署。

工作方式：
1. 先定位相关文件和复现步骤。
2. 能运行测试时先运行最小相关测试。
3. 如果发现两种以上修复路径，请暂停并用表格列出 tradeoff，等我选择。
4. 修改完成后给出 diff 摘要、测试结果和剩余风险。
5. 任何会修改数据库、删除文件、安装全局依赖、访问 secret、推送远端分支的命令，都必须先请求批准。

如果你已经在用 AI 编程代理，可以把日常任务分成三类：

适合手机发起：bug 初查、日志归纳、测试失败分析、文档草稿、PR review 摘要
适合手机批准：安装依赖、运行耗时测试、选择重构方案、确认 diff 方向
仍适合桌面完成：复杂代码审阅、架构决策、手动调试 UI、处理冲突

这条更新的行业信号也很明确：AI 编程工具正在从“聊天式代码补全”转向“可跨设备监督的后台工程代理”。工程师要提升效率，关键是设计好代理可独立推进的任务边界，而不是把手机当成小屏 IDE。

With Codex inside the ChatGPT mobile app, coding agents are no longer tied to the moment you are sitting at your computer. The useful shift for Claude Code / Codex users is not “coding on a phone,” but using the phone as a remote checkpoint for review, approval, and course correction while the desktop agent keeps working.

  • Best fit: long-running tasks. Let Codex run tests, edit files, and prepare diffs on your Mac, devbox, or remote environment while you handle approvals, extra context, and direction from your phone.
  • Your local environment stays local: code, credentials, permissions, dependencies, and setup remain on the machine running Codex; the phone receives live state such as terminal output, screenshots, diffs, test results, and approvals.
  • Turn “waiting for AI” into an async workflow: start a bug investigation in the morning, choose a direction from your phone midway, and review the patch when you return to your desk.
  • Claude Code users can copy the same pattern: split tasks so the agent can proceed independently and explicitly tell it to pause only when a real branch decision is needed.
  • Teams should tighten permission boundaries: once approvals can happen from a phone, dangerous commands, secret access, deployment actions, and destructive file operations should become explicit approval points.

A mobile-friendly task prompt you can reuse:

You are working in my local development environment. Investigate issue #123, but do not commit or deploy anything yet.

Working mode:
1. First identify the relevant files and reproduction path.
2. Run the smallest relevant test set when possible.
3. If you find more than one viable fix, pause and show a tradeoff table before continuing.
4. After changes, summarize the diff, test results, and remaining risks.
5. Ask for approval before any command that modifies a database, deletes files, installs global dependencies, accesses secrets, or pushes to a remote branch.

If you already use coding agents, split your work like this:

Good to start from mobile: bug triage, log summarization, test failure analysis, documentation drafts, PR review summaries
Good to approve from mobile: dependency installs, long test runs, refactor direction, diff direction
Still better on desktop: deep code review, architecture decisions, hands-on UI debugging, merge conflict resolution

The broader signal is clear: AI coding tools are moving from chat-based code assistance toward background engineering agents you supervise across devices. The productivity gain comes from designing tasks with clear boundaries, not from treating the phone as a tiny IDE.

Claude Code v2.1.142：后台 agents 可配置，并修复 MCP 与 daemon 稳定性Claude Code v2.1.142 Adds Configurable Background Agents and Fixes MCP / Daemon Stability

升级后先调整 agents 参数与 MCP 超时。

After upgrading, tune agent flags and MCP timeouts first.

Claude Code v2.1.142 主要增强 claude agents 后台会话的可配置性，并修复 MCP 超时、worktree、daemon、插件和跨平台稳定性问题。对日常使用 Claude Code 的工程师来说，这版值得升级，尤其是依赖后台 agents、MCP 工具或自定义插件的团队。

  • claude agents 新增一组用于分发后台会话的配置 flags：--add-dir、--settings、--mcp-config、--plugin-dir、--permission-mode、--model、--effort、--dangerously-skip-permissions。这意味着你可以把后台 agent 的工作目录、设置、MCP、插件、模型和推理强度显式写进启动命令，而不是依赖交互式环境。
  • 可把常用后台任务封装成 shell alias 或脚本，尤其适合“让 agent 在旁边跑测试修复 / 文档更新 / issue 初筛”：
claude agents \
  --add-dir ../shared \
  --settings .claude/settings.json \
  --mcp-config .claude/mcp.json \
  --plugin-dir .claude/plugins \
  --permission-mode <mode> \
  --model <model> \
  --effort <effort>
  • Fast mode 默认模型从 Opus 4.6 切到 Opus 4.7。升级后如果你发现成本、延迟或输出风格影响了既有自动化，可以临时 pin 回旧默认：
export CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1
claude
  • 插件如果只有根目录 SKILL.md、没有 skills/ 子目录，现在也会作为 skill 暴露。小型个人插件可以更轻：
my-plugin/
  plugin.json
  SKILL.md
  • /plugin 详情面板和 claude plugin details 会展示插件提供的 LSP servers。排查“插件到底有没有提供语言服务”时，先跑：
claude plugin details <plugin-name>
  • MCP_TOOL_TIMEOUT 对远程 HTTP / SSE MCP server 的 per-request fetch timeout 修复后，不再被固定卡在 60 秒。对慢查询、企业内网工具、长耗时检索 MCP 很关键：
export MCP_TOOL_TIMEOUT=180000
claude
  • 后台 sessions 的稳定性修复覆盖多个高频痛点：识别已有 git worktree、macOS sleep/wake 后 daemon 重连、brew upgrade 后旧 binary 路径导致 crash-loop、Chrome extension 连接状态、Windows network-drive 启动死锁、Apple Terminal 256-color 背景色显示、attached session 中点击链接、$EDITOR / $VISUAL 选择。
  • claude --bg --dangerously-skip-permissions 的权限跳过状态在 retire/wake 后会正确保留；但这个模式仍应只用于受信任、可回滚的本地任务。
  • hook 配置错误更明确：如果你给 SessionStart、Setup、SubagentStart 配了 prompt-type 或 agent-type hook，现在会提示应使用 command-type hook。团队可以借此清理 .claude/settings.json 里的 hook 类型。
  • reactive compaction 改进会减少一次接近满上下文时的无效重试；长任务里更少因为压缩时机浪费一轮。

现在能做的一步：升级到 v2.1.142 后，优先检查三件事：后台 agent 启动脚本是否要补 --mcp-config / --plugin-dir，慢 MCP 是否要设置 MCP_TOOL_TIMEOUT，以及 fast mode 自动化是否需要临时设置 CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1。

Claude Code v2.1.142 mainly improves configurability for claude agents background sessions and fixes MCP timeout, worktree, daemon, plugin, and cross-platform stability issues. For daily Claude Code users, this is a worthwhile upgrade, especially for teams relying on background agents, MCP tools, or custom plugins.

  • claude agents adds a new set of flags for dispatched background sessions: --add-dir, --settings, --mcp-config, --plugin-dir, --permission-mode, --model, --effort, and --dangerously-skip-permissions.
