Claude Code RCE 模式：把 deeplink 和 CLI 参数当成不可信输入Claude Code RCE Pattern: Treat Deeplinks and CLI Args as Untrusted Code Paths

RCE 复现提醒：所有 deeplink 和 CLI 参数都要结构化解析。

The RCE write-up is a reminder to parse every deeplink and CLI argument structurally.

这篇复现的是 Claude Code 2.1.118 中与 claude-cli:// deeplink 参数解析相关的 RCE 模式，后续版本已修复。对 AI 编程工具用户，重点不是八卦漏洞，而是学会审计所有「从 URL、prompt、配置文件进入 CLI 参数或 shell」的路径。

  • 典型危险模式：先把 deeplink 或用户输入当字符串扫一遍，再拼成 CLI 参数；如果只用 startsWith("--settings") 这类逻辑，很容易被嵌套参数、编码、重复 flag 或值中的 --flag 绕过。
  • 对 Claude Code 用户：确认版本不停留在受影响的旧版本；不要随手点击未知 claude-cli:// 链接；不要在不可信仓库里使用过宽的自动批准模式。
  • 对自研 AI 工具作者：deeplink、MCP 参数、hook 配置、AGENTS.md / CLAUDE.md 里的命令片段都应视为不可信输入。
  • 安全边界要放在解析之后：先用结构化 parser 得到参数 AST，再做 allowlist 校验；不要在原始字符串层面判断「看起来没有危险 flag」。
  • RCE 类问题尤其要防「配置注入」：攻击者不一定直接注入 rm -rf，也可能注入 --settings、--config、--plugin、--hook，让工具加载恶意配置。

先做版本与仓库检查：

claude --version
claude update

# 搜索仓库里是否存在会自动触发 CLI/deeplink/setting 的片段
rg -n "claude-cli://|--settings|--config|hooks|bypassPermissions|dangerously|shell" .

如果你在写自己的 deeplink handler，避免这种伪安全写法：

// Bad: raw string scanning can be bypassed by encoding or nested values.
if (rawUrl.includes("--settings")) {
  throw new Error("blocked");
}
runCli(rawUrl.split(" "));

改成结构化解析 + allowlist：

import { spawnFile } from "node:child_process";

const allowedParams = new Set(["repo", "branch", "prefill"]);

function parseClaudeLink(rawUrl) {
  const url = new URL(rawUrl);
  if (url.protocol !== "claude-cli:") throw new Error("invalid protocol");

  const args = [];
  for (const [key, value] of url.searchParams.entries()) {
    if (!allowedParams.has(key)) throw new Error(`unsupported param: ${key}`);
    if (value.includes("\0")) throw new Error("invalid value");
    args.push(`--${key}`, value);
  }
  return args;
}

const args = parseClaudeLink(input);
spawnFile("claude", args, { stdio: "inherit", shell: false });

The article reproduces an RCE pattern in Claude Code 2.1.118 related to claude-cli:// deeplink argument parsing, which was fixed in later versions. For AI coding tool users, the lesson is to audit every path where a URL, prompt, or config file becomes CLI arguments or shell execution.

  • Typical dangerous pattern: scan a deeplink or user input as a raw string, then construct CLI args; logic like startsWith("--settings") is easy to bypass with nested parameters, encoding, duplicate flags, or values containing --flag.
  • For Claude Code users: make sure you are not pinned to an affected old build; do not click unknown claude-cli:// links casually; avoid broad auto-approval modes in untrusted repositories.
  • For AI tool builders: deeplinks, MCP parameters, hook configs, and command snippets in AGENTS.md / CLAUDE.md should all be treated as untrusted input.
  • Put the security boundary after parsing: build an argument AST with a structured parser, then apply allowlist validation; do not decide safety from the raw string.
  • RCE often arrives as config injection: the attacker may not inject rm -rf directly, but may inject --settings, --config, --plugin, or --hook so the tool loads malicious configuration.

Codex 锁屏续跑让远程监督更实用。

Codex locked use makes remote supervision more practical.

Codex 的 locked computer use 让 Mac 锁屏后仍可继续执行受信任的 computer-use 任务，并能配合 Codex Mobile 远程监督。对长时间跑测试、修 UI、操作桌面 app 的 Codex 用户，这是从“人必须守在电脑前”转向“移动端审批 + 后台执行”的重要体验升级。

  • 适合场景：长测试矩阵、Xcode / Simulator 调试、浏览器或桌面 app 操作、需要中途人工批准的多步骤任务。
  • 不适合场景：合上 MacBook 盖子后的睡眠模式、无人监督的敏感账号操作、涉及生产凭据或不可逆提交的桌面流程。
  • 开启后仍要保持最小权限：只在可信 repo、可信任务、可信网络环境中使用；不要把 locked use 当成通用远程解锁通道。
  • 最佳组合是 Goal mode + 明确验收标准 + mobile approval：让 Codex 长时间推进目标，但在危险动作前停下来等你确认。
  • 如果只是 terminal 长任务，仍可用 caffeinate 防止睡眠；locked computer use 的价值在于 GUI / desktop control 能在锁屏后继续。

Goal: Fix the failing checkout flow in the macOS app and verify it in Simulator.
Success criteria:
1. Reproduce the failing flow.
2. Patch the smallest root cause.
3. Run the focused unit/UI tests.
4. Capture evidence of the fixed screen.
5. Stop before changing signing, credentials, production endpoints, or App Store settings.

While running remotely or while the Mac is locked:
- Ask before submitting forms, deleting data, changing accounts, or installing dependencies.
- Report every command and desktop action that changes state.
- If blocked by permissions or a locked credential prompt, stop and wait.
