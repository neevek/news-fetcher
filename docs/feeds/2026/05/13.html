Show HN：`mi v1.7.5` 用 `--sandbox` 在 Docker 中运行轻量 agent loopShow HN: Tiny agentic loop with Docker sandbox

Docker sandbox 是本地 agent 循环的实用安全层。

mi v1.7.5 的亮点是新增 --sandbox，把 agent 放进 Docker 中运行，并挂载当前目录以限制意外操作的影响范围。对本地 agent 用户来说，这是一个很小但实用的安全模式参考。

  • mi 支持 OpenAI-compatible model，包括本地模型；适合实验轻量 agent loop、REPL、skills 和 recursive agents。
  • --sandbox 的价值在于降低 blast radius：agent 能操作项目文件，但系统级影响被 Docker 环境隔离。
  • 适合运行“修测试直到通过”这类自动循环任务，尤其是在你不完全信任模型或 prompt 可能触发意外命令时。
  • 示例用法：
mi --sandbox -p 'fix tests until all are passing'
  • 对 Codex/Claude Code 的借鉴：即使工具本身有权限提示，也可以用 Docker、git worktree、只读 secrets 等外层隔离再加一道边界。

New in mi v1.7.5: "--sandbox" runs the agent inside Docker with your current directory mounted, to contain the blast radious in case of unexpected attacks.

mi Works with any OpenAI-compatible model (including local ones). Has REPL, skills, and recursive agents and only 30 lines of code, optimised for local models.
