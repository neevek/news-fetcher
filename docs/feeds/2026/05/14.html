OpenAI：如何安全运行 Codex 并控制 agent 风险Running Codex Safely at OpenAI

Codex 安全运行的核心是权限边界和审批策略。

OpenAI 的“Running Codex Safely”聚焦 Codex 的安全运行边界：让 coding agent 能执行真实开发任务，同时降低命令、文件、网络和凭据带来的风险。对工程师来说，这是设计本地 AI 编程工作流时必须补上的安全层。

  • 给 Codex 明确权限范围：允许读写哪些目录、哪些命令需要审批、哪些操作必须禁止。
  • 把高风险命令单独处理：删除文件、修改权限、访问密钥、执行迁移、推送代码、安装未知脚本都应进入人工审批。
  • 优先使用 sandbox、临时 worktree 或容器运行不可信任务，避免 agent 直接操作主工作区。
  • 让模型先解释再执行：要求它在运行命令前说明目的、预期结果、失败处理和影响范围。
  • 现在可做的一步：为 Codex 建一个专用工作流，默认在新 branch 或 git worktree 中运行，并要求所有 destructive commands 先请求批准。

Show HN：`1-800-CODER` 用语音和指向操作让 AI 修改网页Show HN: 1-800-CODER, macOS app where you call an AI developer to edit your page

语音加指向交互能提升前端微调效率。

1-800-CODER 是一个 macOS voice coding agent demo，基于 OpenAI gpt-realtime-2，让用户像和远程开发者通话一样修改网页。它对 Claude Code/Codex 用户的启发在于：语音 + 指向界面元素可以显著提升 UI 修改任务的沟通带宽。

  • 适合前端微调场景：例如“把这个 hero 压紧一点”“删掉这个模块”“让那个标题加粗”，语音比长 prompt 更接近设计评审。
  • “pointing” 是关键交互：用户指向页面元素后再描述意图，能减少选择器、组件名、DOM 结构的解释成本。
  • 对现有 agent 的借鉴：在浏览器预览、截图、DOM inspection、视觉定位之间建立映射，让模型能把自然语言指令落到具体元素。
  • 不适合替代复杂架构改造；更适合 landing page、营销页、样式 polish、文案布局等高频小改动。
  • 现在可做的一步：在使用 Codex/Claude Code 做 UI 调整时，附上截图并用明确位置描述，比如“右上角 CTA”“第二个 pricing card”，减少模型误改。

Sharing a small Mac app I built around OpenAI’s gpt-realtime-2 model. You call up a voice coding agent and talk to it like you’d talk to a freelancer ("make the hero tighter, put a product image on the right, that one's too big"). You can even point at things on your webpage and say “remove this” or “make that bold”. Pointing feels like a killer feature. It pushes the conversation bandwidth way up, and feels just like working with a real person over a screen share.
