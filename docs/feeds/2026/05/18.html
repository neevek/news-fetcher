这类关于低价 LLM 是否可持续的讨论，对工程师的实际价值是提醒你不要把 AI 编程生产力完全绑定在单一价格假设上。

  • 生产工作流应按任务分层：高价值设计、复杂调试用强模型；机械改名、格式化、简单测试修复可用便宜模型或本地模型。
  • 把 prompts、AGENTS.md、评审清单和测试命令沉淀下来，可以降低模型切换成本；工具变贵时，流程资产仍能复用。
  • 对 Claude Code / Codex，建议把大任务拆成可验证小步，避免一次超长上下文消耗；优先让 agent 运行测试和读取具体文件，而不是泛泛扫描整个仓库。
  • 现在能做的一步：给团队定义三档模型策略：fast、standard、deep，并写清每档适用的任务类型和成本上限。

This discussion about whether low-cost LLMs are temporary is useful because it reminds engineers not to anchor AI coding productivity on a single pricing assumption.

  • Tier production workflows by task: use strong models for high-value design and complex debugging; use cheaper or local models for mechanical renames, formatting, and simple test fixes.
  • Preserve prompts, AGENTS.md, review checklists, and test commands so model switching costs stay low; if tools get more expensive, workflow assets still carry over.
  • For Claude Code / Codex, break large tasks into verifiable steps to avoid long-context waste; have agents run tests and inspect specific files instead of broadly scanning entire repositories.
  • One step to take now: define three model tiers for your team: fast, standard, and deep, with task types and cost ceilings for each.
